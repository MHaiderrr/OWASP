A02: Cryptographic Failures

What are Cryptographic Failures?

Cryptographic Failures occur when sensitive data is not properly protected through encryption, leading to unauthorized access. This includes using weak or outdated cryptographic algorithms, improper key management, and failing to encrypt sensitive data altogether.

Common Exploitation Techniques

  • Exploiting weak encryption algorithms (e.g., MD5, SHA-1).
  • Intercepting data transmitted without encryption (e.g., HTTP instead of HTTPS).
  • Retrieving hard-coded cryptographic keys from application code.
  • Performing padding oracle attacks due to improper implementation.

Tools for Practice

  • Hashcat: A robust password recovery tool for cracking hashed passwords.
  • John the Ripper: A fast password cracker to detect weak passwords.
  • CyberChef: A web-based tool for encryption, encoding, and data analysis.

Steps to Learn and Practice

  1. Study modern cryptographic algorithms and their proper implementations.
  2. Learn to identify weak or deprecated encryption methods in applications.
  3. Practice decrypting data encrypted with weak algorithms using tools like Hashcat.
  4. Understand best practices for key management and data encryption.
