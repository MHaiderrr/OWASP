A08: Software and Data Integrity Failures

What are Software and Data Integrity Failures?

Software and data integrity failures occur when an application does not adequately verify the integrity of its software and data. This can lead to unauthorized modifications, data corruption, and exploitation of vulnerabilities within the software supply chain.

Common Types of Software and Data Integrity Failures

  • Insufficient Verification of Data Authenticity: Failing to verify the source of data can allow malicious data to be processed.
  • Missing Support for Integrity Checks: Lack of checks can enable attackers to modify or delete data without detection.
  • Untrusted Search Path: Allowing attackers to control the search path for libraries can lead to code injection.
  • Download of Code Without Integrity Check: Downloading code without verifying its integrity can introduce malicious software.
  • Deserialization of Untrusted Data: Improper handling of serialized data can allow attackers to execute arbitrary code.

Real-Life Example

The SolarWinds attack in 2020 is a significant example where a compromised software update propagated malicious code to numerous organizations, highlighting the risks associated with software and data integrity failures.

Tools for Practice

  • Nessus: A vulnerability scanner that helps identify integrity issues in your applications.
  • OWASP Dependency-Check: A tool that identifies known vulnerabilities in project dependencies.
  • Snyk: A tool focused on finding and fixing vulnerabilities in open source dependencies.

Steps to Learn and Practice

  1. Understand the principles of software integrity and its importance in application security.
  2. Learn about common vulnerabilities related to software updates and supply chain attacks.
  3. Practice using tools like OWASP Dependency-Check to identify vulnerable components.
  4. Implement secure coding practices to prevent integrity failures in your applications.
